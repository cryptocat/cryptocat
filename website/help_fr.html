Informations importantes sur les versions bêta

Cryptocat est en ce moment (mars 2016), un logiciel en version bêta. Cela signifie qu'il est en phase de pré-publication : c'est le moment où nous invitons le grand public à tester l'application et à faire remonter des bugs. Pendant cette période, nous vous recommandons vivement de prendre un minimum de précautions quand vous utilisez Cryptocat, car il subit une série de tests avant de passer en version « stable ».

Si vous rencontrez un problème avec votre client Cryptocat et que vous souhaitez nous en faire part, veuillez le faire en utilisant notre suivi de bugs sur GitHub.

Créer un compte

Vous pouvez créer un compte avec la fenêtre d'identification de Cryptocat, en cliquant sur le bouton Créer un compte.
Vous pouvez également modifier le mot de passe de votre compte en cliquant sur Compte > Changer le mot de passe dans la barre de menu de Cryptocat.

Ajouter et supprimer des appareils

Quand vous vous connectez pour la première fois à votre compte Cryptocat avec un nouvel appareil, il vous est demandé de préciser quel est cet appareil et de l'associer à votre compte. Notez que ceci va générer et stocker des clés de chiffrement sensibles sur votre appareil ; consultez Gérer vos appareils en toute sécurité pour en savoir plus sur la gestion des données sensibles.

Une fois que vous avez ajouté un nouvel appareil, vous n'aurez pas la possibilité de changer son nom ni son icône par la suite. Pour afficher vos appareils à tout moment, il vous suffit d'ouvrir le menu Compte et de cliquer sur Gérer les appareils, ou d'appuyer sur Alt+D. dans le gestionnaire d'appareils, vous aurez aussi la possibilité de supprimer des appareils associés à votre compte Cryptocat. Deux choses peuvent se produire quand vous supprimez un appareil :

  • Si vous supprimez l'appareil que vous utilisez justement à ce moment : vous allez être déconnecté et les informations sensibles sur cet appareil seront supprimées sur celui-ci.
  • Si vous supprimez un autre appareil que celui que vous utilisez à ce moment : l'appareil sera dissocié de votre compte, mais les informations sensibles sur cet appareil demeureront sur celui-ci. Si vous voulez, vous pouvez supprimer ces données manuellement sur l'appareil lui-même.

Vérifier les empreintes de votre interlocuteur

De façon à obtenir un maximum de confiance dans les appareils utilisés par votre correspondant, vous pouvez vérifier ses empreintes numériques. Il s'agit d'un identifiant spécial pour chaque appareil particulier. Cela ressemble à ça : 99:37:8c:a1:32:05:c5:97:ac:f9:32:d4:18:1e:3e:df.

Par exemple, vous pouvez voir les appareils de Florent en cliquant droit sur votre liste de contacts et en sélectionnant Voir les appareils. Vous pourrez alors voir que votre ami Florent a un appareil appelé « ordi de Florent ». Mais comment pouvez-vous avoir la certitude que c'est bien l'ordinateur de Florent et non un appareil frauduleusement ajouté dans une intention malveillante ? Voici comment faire : vous pouvez appeler Florent au téléphone ou le rencontrer dans la vraie vie, pour lui demander de vous lire le sempreintes numériques de ses appareils. Si elles coïncident avec ce que vous pouvez voir dans votre fenêtre Voir les appareils, c'est que vous avez pu authentifier avec succès les appareils de Florent. À partir de cet instant, quand vous envoyez un message à Florent, vous pouvez être assuré qu'il sera bien reçu par les appareils dont Florent a déclaré qu'ils étaient les siens sur son compte.

Si un de vos amis ajoute un nouvel appareil à son compte, Cryptocat vous en informera. Nous vous recommandons de vérifier les appareils avec vos correspondants. C'est une précaution que vous n'aurez à prendre qu'une seule fois par appareil et cela vous protègera contre les risques d'une imposture dangereuse dans les conversations que vous menez.

Gérer les appareils en toute sécurité

Quand vous faites une association d'un nouvel appareil à votre compte Cryptocat, l'application va générer des clés de chiffrement sensibles et les stocker sur ce nouvel appareil. Il est important de bien comprendre que ces informations hautement sensibles ne sont pas protégées par quoi que ce soit et qu'un pirate malveillant qui accède à votre appareil peut récolter ces informations.

En-dehors d'informations peu sensibles telles que vos paramètres de notification, les informations concernant votre appareil comprennent les détails sensibles suivants :
  • Vos clés d'identité : un voleur qui accède à ces clés peut simuler l'usage de cet appareil par la suite, mais ne pourra pas déchiffrer vos conversations passées sur Cryptocat, grâce à la fonctionnalité avancée de confidentialité de Cryptocat.
  • Votre nom d'utilisateur : un voleur pourra apprendre votre nom d'utilisateur pour Cryptocat.
  • Votre liste de contacts : un voleur pourra connaître le contenu de votre liste de contacts pour Cryptocat.
Les informations concernant votre appareil sont stockées dans :
  • Windows: C:\Users\(Your Username)\AppData\Roaming\Cryptocat\users.db
  • Linux: ~/.config/Cryptocat/users.db
  • Mac: ~/.config/Cryptocat/users.db

Pourquoi les informations sur l'appareil ne sont-elles pas chiffrées ?

Il n'est pas évident qu'il y aurait un gros avantage à ce que Cryptocat lui-même chiffre les informations concernant votre appareil. Et demander aux utilisateurs d'utiliser un autre mot de passe, différent pour chaque appareil en plus de celui du compte Cryptocat peut rapidement s'avérer une impasse en termes d'ergonomie pour la vaste majorité d'utilisateurs. le chiffrement des informations sur l'appareil pourrait apporter davanatge de sécurité, mais nous estimons que cette sécurité supplémentaire est superficielle : tout ce qu'un attaquant qui sait exécuter du code a besoin de faire, c'est d'attendre que vous soyez connecté à Cryptocat et de charger en mémoire vos informations déchiffrées sur l'appareil avant d'attaquer. C'est pourquoi il ne nous semble pas que ce soit une solution viable qui justifierait qu'on demande aux utilisateurs de se souvenir de plusieurs mots de passe pour le même compte.

Pour le dire de façon brutale, si vous n'avez pas confiance dans votre appareil, Cryptocat ne peut pas faire grand-chose pour vous sur ce point.

Que puis-je faire pour augmenter la protection des informations sur mon appareil ?

Utilisez le chiffrement total de votre disque dur. Sous Windows, vous pouvez activer le chiffrement de l'appareil. Sous Linux, vous pouvez utiliser LUKS, dm-crypt ou eCryptfs. Sous Mac, vous pouvez utiliser FileVault. Ces mesures vous assureront que tous les fichiers de votre ordinateur seront mieux protégés. Les informations sur votre appareil seront plus difficiles à récupérer si votre appareil est éteint au moment où il vous est volé.

Activer/désactiver les alertes sonores et les notifications

Par défaut, les notifications sur votre ordinateur et les alertes sonores sont activées dans Cryptocat, de façon à réduire les risques pour vous de rater un message important. Certains utilisateurs peuvent trouver que c'est pénible et choisir de les désactiver. Voici comment faire :

  • Windows et Linux : cliquez droit sur l'icône de Cryptocat dans la zone de notification de votre système. Un menu va apparaître, il vous permet de cocher ou décocher les paramètres Alertes sonores et Notifications.
  • Mac : cliquez droit sur l'icône de Cryptocat dans votre dock. Dans le menu contextuel quipparaît vous pouvez cocher ou décocher les options de notifications et alertes sonores.

Vérifier l'intégrité du téléchargement de Cryptocat

Tous les clients Cryptocat sont signés de sorte que vous pouvez en vérifier l'authenticité. Cela signifie que quand vous téléchargez un client Cryptocat, vous pouvez vérifier qu'il vient bien du développeur de Cryptocat. Cela contribue à empêcher les gens malintentionnés de vous tromper en vous faisant télécharger une version contrefaite du client.

Remarque à propos des mises à jour : si vous téléchargez une mise à jour de Cryptocat en utilisant le gestionnaire intérgré de mises à jour, la vérification de signature numérique est effectuée automatiquement. Vous devrez toutefois effectuer les vérifications dont les étapes sont décrites ci-dessous.

Vérifier Cryptocat pour Windows

Cliquez droit sur Setup.exe et sélectionnez Propriétés. dans la fenêtre des propriétés, cliquez sur Signatures numériques. Il devrait y avoir une signature dont les détails coïncideront avec ce qui suit :

  • Nom: Open Source Developer, Nadim Kobeissi
  • Empreinte: 17 2b 46 56 59 d0 d2 a2 45 c8 d0 fb 70 ce e3 81 76 2e 21 a5

Vérifier Cryptocat pour Linux

Après avoir décompressé votre téléchargement, vous pouvez vérifier son authenticité en lançant la commande gpg2 --verify Cryptocat.zip.asc Cryptocat.zip dans votre terminal. Le résultat devrait ressembler à ceci : gpg: Signature made Sat 10 Apr 2016 12:34:56 PM CET using EDDSA key ID 5D93C3CB
gpg: Good signature from "Nadim Kobeissi (Cryptocat Client Signing Key) <nadim@nadim.computer>"

Vérifier Cryptocat pour Mac

Après l'ouverture de Cryptocat.pkg, cliquez sur l'icône de cadenas dans l'angle supérieur droit de la fenêtre de l'installeur. Vous devriez voir apparaître un certificat conforme à celui-ci :

  • Nom : Developer ID Installer: Nadim Kobeissi (HC689Z8JM4)
  • Empreinte (SHA1): DE 35 C1 E8 0E 36 09 C5 DB E3 CF F9 01 88 BF 9B EB F2 47 1D
Vous pouvez aussi vérifier l'installeur sans l'ouvrir d'abord en utilisant cette ligne de commande : pkgutil --check-signature Cryptocat.pkg.
